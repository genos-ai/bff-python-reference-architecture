# Documentation

All project documentation, organized by purpose.

## Structure

| Directory | Purpose |
|-----------|---------|
| `01-getting-started/` | Installation, setup, quickstart guides |
| `02-architecture/` | System design, diagrams, architecture decisions |
| `03-principles/` | Design principles and architectural mandates |
| `04-rules/` | Machine-readable compliance rules (JSONL) |
| `05-plans/` | Implementation plans and progress tracking |
| `98-research/` | Research documents, external references, deep dives |
| `99-reference-architecture/` | Authoritative architecture standards (core + optional) |

## Naming Convention

Documents within each directory use zero-padded numbering:

```
001-topic-name.md
002-topic-name.md
```

This ensures correct sort order and makes it easy to reference documents by number.
