# Principles

Design principles and architectural mandates that govern all decisions.

## Naming Convention

```
001-backend-first.md
002-thin-client-mandate.md
003-simplicity-over-cleverness.md
```

## Adding a Principle

Create a new file with the next available number. Each principle document should include:

- **Statement** — one-sentence principle
- **Rationale** — why this principle exists
- **Implications** — what this means in practice
- **Examples** — concrete code or design examples
